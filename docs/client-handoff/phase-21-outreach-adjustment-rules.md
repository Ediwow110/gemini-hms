# Phase 21: Outreach Adjustment Decision Rules

Use these rules to determine when to scale, adjust, or stop current outreach efforts.

| Condition | Observation | Decision |
|-----------|-------------|----------|
| **High Fit / High Response** | Reply rate > 10%; Fit Score > 24. | **SCALE**: Proceed to next batch. |
| **Low Response** | Reply rate < 2% across 20+ messages. | **ADJUST**: Review value proposition/subject line. |
| **High Opt-Out** | Opt-out rate > 25%. | **PAUSE**: Stop outreach for 48h; review segment fit. |
| **Complaint Received** | Direct complaint about outreach method. | **REVIEW**: Immediate escalate to Project Owner. |
| **PHI Request Demand** | Prospect insists on real data trial. | **STOP**: Disqualify for current phase; explain policy. |
| **No Budget for Staging** | Prospect refuses $50/mo cloud cost. | **DISQUALIFY**: Do not offer demo; keep as "nurture." |
| **Security Escalation** | technical vulnerability reported. | **PRIORITIZE**: Cease outreach; assign to Cybersecurity lead. |
| **Staging Trigger Met** | All 8 Phase 21-0 criteria confirmed. | **INITIATE**: Move to Phase 21 Implementation. |

## Governance Process
1. **Inputs**: Reply quality, opt-out rate, complaint count, fit score data.
2. **Review Cycle**: Every 50 messages or every 7 days (whichever is sooner).
3. **Approval**: Adjustments to message variants must be reviewed for truthfulness by the Product Lead.
