# Phase 21: Batch 1 Final Send Checklist

**Mandatory**: This checklist must be completed by a human operator before sending any outreach from Batch 1.

## 1. Research & Contact Verification
- [ ] **Public Origin**: Verified the organization name and public URL are from an official source.
- [ ] **Official Inbox**: Using an official info@, admin@, or web contact form only.
- [ ] **No Private Data**: Confirmed NO personal email addresses or cell numbers are used.
- [ ] **Zero PHI**: Verified that no patient information is included in the message or research notes.

## 2. Technical Accuracy (The 4 Boundaries)
- [ ] **"Local Green"**: Message states the baseline is locally verified/development state.
- [ ] **"Synthetic Only"**: Message explicitly states demos use synthetic data.
- [ ] **"Audit Pending"**: No claims of active HIPAA or SOC2 certification are made.
- [ ] **"Not Live"**: No claims that the system is currently live on a public cloud.

## 3. Personalization Ethics
- [ ] **Truthful News**: Only mentioned "following your news" if a specific public fact was researched.
- [ ] **Attachment Integrity**: If "I’ve attached" is used, the approved PDF/Doc is actually included.
- [ ] **No Deceptive Identity**: Operator is identifying correctly as a representative of the project.

## 4. Operational Governance
- [ ] **Opt-Out Statement**: Message includes a clear, respectful way for the recipient to opt out.
- [ ] **Tracker Update**: Status in `phase-21-batch-1-prospect-table.md` set to READY.
- [ ] **Escalation Plan**: Operator has read `phase-21-safety-escalation-playbook.md` for potential PHI requests.

## 5. Send Conditions
- [ ] **Final Approval**: Explicit approval to send has been granted by the Project Owner.
- [ ] **Response Package Ready**: `phase-21-response-templates.md` and matrix are available for follow-up.
