# Screenshot Checklist

This checklist defines the visual proof required for a complete client demo package. **Note: Use only synthetic/demo data when generating these images.**

### 1. Authentication & Access
- [ ] **Login Screen**: Showing staff and patient login options.
- [ ] **MFA Challenge**: Showing the secondary security layer.
- [ ] **Role Isolation**: Screenshot of sidebar for `Doctor` vs `Pharmacist`.

### 2. Clinical Workflow
- [ ] **Patient Queue**: List of patients pending vitals or triage.
- [ ] **EMR Workspace**: A completed SOAP note in draft state.
- [ ] **Clinical Orders**: List of ordered labs and medications.

### 3. Lab Workflow
- [ ] **Result Encoding**: The form used to input lab values.
- [ ] **Release View**: The manager's view for final lab approval.

### 4. Pharmacy Hub
- [ ] **Dispense Queue**: Active prescriptions awaiting the pharmacist.
- [ ] **Drug Inventory**: View of current stock levels.

### 5. Billing & Admin
- [ ] **Billing Overview**: Invoice status and payment history.
- [ ] **Audit Log Viewer**: The forensic trail showing clinical updates.
- [ ] **Tenant Management**: Multi-clinic administration view.

---
**Disclaimer**: Do not use real patient names or data when capturing these images. Ensure "DEMO DATA" watermark or clear labeling is visible if possible.
